**Phase 8 — `compute_disorder` metric (subject VI.3.2)**

- Implemented `compute_disorder(a)` on `src/algorithms/adaptive/compute_disorder.c`. Returns the inversion ratio (disorder) in [0, 1]: counts pairs where an earlier element is greater than a later one, divided by C(n,2).
- **Design decision — compare by value, not index.** Subject's pseudo-code uses `a[i] > a[j]` (value). Comparing by index gives an identical result (coord_compress is order-preserving), but using value matches the pseudo-code literally and sidesteps any "measure disorder before any moves" interpretation debate — disorder is just a comparison, it doesn't need the index's bit properties. Confirmed the subject explicitly bans nothing here, and coord_compress emits zero ops so "before any moves" is satisfied either way.
- Two bugs caught at review:
  1. **Integer division**: I first wrote `(double)(mistakes / total_pairs)` — the int division runs first and truncates e.g. 1/3 to 0, so disorder only ever returned 0 or 1. Fixed to `(double)mistakes / total_pairs` (cast before dividing).
  2. **Error-path cleanup** in main.c had `stack_init(&stack_b)` where `stack_free` was meant.
- Verified: `1 2 3 → 0.0`, `3 2 1 → 1.0`, `2 1 3 → 0.333`, `5 8 -4 -3 1 → 0.6`, `42 → 0.0`. Hand-checked the 0.6 case (6 inversions / 10 pairs).
