# Work Log / 作業ログ

## English

Daily log of what each partner worked on, what's next, and what's blocking.
Both **So** and **Javier** write their own entries; one entry per partner per day.

### File naming

- **Format**: `YYYY-MM-DD_<author>.md`
- **Authors**: `so` (Souta) and `javi` (Javier)
- **Examples**: `2026-05-28_so.md`, `2026-05-28_javi.md`
- **Rationale**: date keeps chronological sort; suffix disambiguates parallel entries from the two partners

Use `TEMPLATE.md` as a starting point for each new entry.

### Purpose

- Self-reflection and continuity (pick up where left off)
- Visible contribution record for the pair defense (subject VI.1)
- Surface blockers early to the partner

### Tip

Keep entries short and honest. "Spent 2h debugging rotate, still stuck" is more useful than "implemented rotate".

## 日本語

両者が日々何をしたか、次に何をするか、何で詰まっているかを記録する場所です。
**So** と **Javier** がそれぞれ自分のエントリを書きます(1 人 1 日 1 ファイル)。

### ファイル命名

- **形式**:`YYYY-MM-DD_<author>.md`
- **author 略名**:`so`(草壁)、`javi`(Javier)
- **例**:`2026-05-28_so.md`、`2026-05-28_javi.md`
- **根拠**:日付を先頭にすることで時系列ソートを維持、suffix で両者の並行エントリを識別

新規エントリは `TEMPLATE.md` をコピーして使ってください。

### 目的

- 振り返り・継続性(中断後に再開しやすく)
- ペア課題ディフェンス用の貢献記録(subject VI.1)
- 詰まり事項を相方に早期共有

### コツ

**短く・正直に**。「rotate のデバッグに 2 時間、まだ未解決」のほうが「rotate を実装した」より有用。
