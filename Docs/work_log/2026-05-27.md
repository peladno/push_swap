**M1 — debugging and fixes**
Initial Makefile had multiple issues caught during review:
- `NAME = push_swap.a` → should be `push_swap` (executable, not archive).
- `ar rcs` (libft style) → should be `$(CC) -o $@ $(OBJS)` (linker for executable).
- `HEADER = push_swap.h` referenced a non-existent file at repo root; the actual path is `include/push_swap.h`. As a result, the custom pattern rule was silently ignored and Make fell back to its built-in rule (visible from the compile command's argument order).
- `CFLAGS` was missing `-I include`, which would break as soon as `main.c` included the header.
- `main.c` triggered `-Wunused-parameter` because `argc`/`argv` were declared but unused.

After fixes:
- All make targets work (`make`, `clean`, `fclean`, `re`).
- Custom pattern rule is now used (verifiable from compile command order: `-c $< -o $@`).
- Subject II compliance verified: second `make` produces `Nothing to be done for 'all'` (no unnecessary relinking).

**Makefile concepts review (study session)**
Walked through:

- `$@` and `$<` — automatic variables in rules (target / first prerequisite); enables generic pattern rules.
- `-I <path>` — compiler include search path; clarified that `#include "..."` searches first in **the source file's own directory**, not `pwd`. With headers in `include/` separate from sources in `src/`, `-I include` is required.
